# Zero Venture Operating Framework

## Purpose

Convert any idea, client, vertical, or project into a repeatable process for research, diagnosis, offer design, building, selling, delivery, and accumulated learning.

## Core Principle

Do not build isolated projects. Build reusable assets.

Every project should produce at least one asset:

1. Playbook
2. Tool
3. Dataset
4. Landing
5. Prompt
6. Report
7. Case study
8. Sellable offer
9. Commercial relationship
10. Reusable automation

## Philosophy: Cart to Formula 1

### Cart

Manual, one-off work with no reuse.

- Research from scratch.
- Build from scratch.
- Propose from scratch.
- Sell from scratch.
- Learn, but do not systematize.

### Automatic Cart

AI makes manual work faster, but every project is still independent.

### Formula 1

Build a machine that improves with every project.

- Every research project improves the research engine.
- Every client improves the diagnosis engine.
- Every proposal improves the offer engine.
- Every delivery improves the delivery engine.
- Every error improves the operating system.

## Decision Rule

Before accepting or advancing any project, Zero asks:

> Does this project improve the central engine, or does it add noise?

A project moves forward only if it satisfies at least 2 criteria:

1. Can generate revenue in less than 90 days.
2. Can become a case study.
3. Improves an internal tool.
4. Opens access to a valuable sector.
5. Produces a repeatable offer.
6. Creates transferable learning.
7. Can be partially automated.
8. Strengthens one of your core ventures or the operating system itself.

If it does not satisfy at least 2, pause it.

## Universal Pipeline

Every project passes through this pipeline:

1. Intake
2. Market Research
3. Company / Sector Diagnosis
4. Opportunity Map
5. Economics Analysis
6. Offer Design
7. MVP / Tool Design
8. Landing / Sales Asset
9. Outreach / Meeting Prep
10. Delivery
11. Case Study
12. System Update

## Required Project Outputs

1. Project Intake Brief
2. Market Intelligence Report
3. Operational Diagnosis
4. Opportunity Map
5. Economics Memo
6. Offer Design
7. MVP Spec
8. Landing / Sales Asset Brief
9. Meeting Prep
10. Delivery Plan
11. Case Study structure
12. System Update

## Final Pre-Build Gate

Before building anything, Zero must produce:

1. What we know.
2. What we do not know.
3. What data is missing.
4. What hypothesis we will validate.
5. How this can make money.
6. What can be sold first.
7. What not to build yet.
8. Go / Explore / Pause / Kill.
