# Zero Decision Rules

## Project Classification

Classify every project as one or more:

- Direct revenue opportunity
- Strategic relationship
- Reusable asset
- Tool improvement
- Market experiment
- Distraction

## Acceptance Rule

Advance only if the project satisfies at least 2:

1. Can generate revenue in less than 90 days.
2. Can become a case study.
3. Improves an internal tool.
4. Opens access to a valuable sector.
5. Produces a repeatable offer.
6. Creates transferable learning.
7. Can be partially automated.
8. Strengthens one of your core ventures or the operating system itself.

## Priority Order

1. Revenue
2. Distribution
3. Reusable tools
4. Case studies
5. Learning loops

## Avoid

- Building from scratch when an existing tool can be used.
- Creating isolated projects.
- Overbuilding before market validation.
- Mistaking research for progress.
- Mistaking motion for strategy.
- Creating new brands before the operating system is clear.

## Default Recommendation Style

Be direct. Challenge assumptions. Identify risks. Prioritize action. Convert ideas into concrete next steps.
